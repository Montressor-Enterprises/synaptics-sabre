Configures DWC_mipi_i3c to have high data rate - Double Data Rate Mode support.

If this parameter is selected, DWC_mipi_i3c transfers the data on both edge transitions of the SCL for generation of high data rate.

Values: 0, 1

DefaultValue: 0

Enabled: Always

Parameter Name: IC_SPEED_HDR_DDR
HDR-TS false

Configures DWC_mipi_i3c to have high data rate - Ternary Symbol Mode support.

If this Parameter is selected, DWC_mipi_i3c transfers the data as Ternary symbols by using both SCL and SDA lines as data lines. DWC_mipi_i3c requires the recovered clock as an external input to sample the data. You can use either CDR block or Pure Digital DPLL solutions to recover the clock. Note that these clock recover mechanisms are not a part of DWC_mipi_i3c deliverables and are expected to be implemented outside the IP boundary.

Values:

  • true (1)

Input Strap for SIR request 0

When Enabled input ports will be made available for requesting for SIR. In this case the user must ensure that the SLV_INTR_REQ register is not going to be accessed through APB. The strap input is expected to be a pulse of one core clock duration. When this pulse input is provided the SIR bit in the SLV_INTR_REQ register will be set. The SIR will be generated from the controller only if the slave logic is out of reset and enabled. Please refer section 3.3 in user guide. The Slave being out of reset and enabled will generate the SIR only after the bus available time programmed in BUS_FREE_AVAIL_TIMING register. NOTE: Preferred mechanism for IBI initiation is through programming the IBI registers(SLV_INTR_REQ).
