APB Slave Interface Signals

Port Name I/O Description

pclk

 I

APB clock.

Synchronous To: None

Registered: N/A

Power Domain: SINGLE_DOMAIN

Active State: N/A

presetn

 I

APB reset signal. Asynchronous APB interface domain reset. This signal resets only the bus interface. The signal is asserted asynchronously, but is deasserted synchronously after the rising edge of pclk. The synchronization must be provided external to this component.

Synchronous To: Asynchronous

Registered: N/A

Power Domain: SINGLE_DOMAIN

Active State: Low

psel

 I

APB peripheral select that lasts for two pclk cycles. When asserted, indicates that the peripheral has been selected for read or write operation.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High

penable

 I

APB enable control. Asserted for a single pclk cycle and used for timing read or write operations.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High

pwrite

 I

APB write control.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: When high, indicates a write access to the peripheral; when low, indicates a read access.

paddr[7:0]

 I

APB address bus. Uses lower 8 bits of the address bus for register decode.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: N/A

pwdata[31:0]

 I

APB write data bus. Driven by the bus master (bridge unit) during write cycles. Can be 8, 16, or 32 bits wide.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: N/A

pready

 O

Indicates whether a request cycle was accepted.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High

pslverr

 O

Flag for the slave error response from APB.

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

prdata[31:0]

 O

APB readback data. Driven by the selected peripheral during read cycles. Can be 8, 16, or 32 bits wide

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: N/A

Serial Interface Signals

Port Name I/O Description

rxd[0:0]

 I

Receive data signal. Input data from a serial-master or serial-slave device is received on this line.

Synchronous To: sclk_in

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: N/A

ss_in_n

 I

Slave select input. When configured as a serial slave, this signal selects the device. When configured as a serial master, this signal can be used to inform the system of master contention on the bus.

Synchronous To: ssi_clk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: Low

ssi_clk

 I

Peripheral serial clock signal.

Synchronous To: None

Registered: N/A

Power Domain: SINGLE_DOMAIN

Active State: N/A

ssi_rst_n

 I

Peripheral serial reset signal.

Synchronous To: Asynchronous

Registered: N/A

Power Domain: SINGLE_DOMAIN

Active State: Low

ssi_sleep

 O

SSI enable flag. This signal is asserted when the DW_apb_ssi is enabled. It can be used by the system clock generator/controller module in order to disable the ssi_clk input of the DW_apb_ssi; this reduces power consumption in the system.

0- Not safe to remove ssi_clk

1- Safe to remove ssi_clk

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

ssi_busy

 O

SSI busy flag. This signal is asserted when the DW_apb_ssi is busy with serial transfer. This signal can be used to know the status of DW_apb_ssi instead of polling for busy bit in SR register.

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

txd[0:0]

 O

Transmit data signal. Output data from the serial master or serial slave is transmitted on this line.

Synchronous To: ssi_clk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: N/A

ssi_oe_n[0:0]

 O

Output enable signal. Used to control external buffers on serial output lines.

Synchronous To: ssi_clk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: Low

DMA Interface Signals

Port Name I/O Description

dma_tx_ack

 I

DMA Transmit Acknowledgement Sent by the DMA Controller to acknowledge the end of each DMA burst or single transaction to the transmit FIFO.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High

dma_rx_ack

 I

DMA Receive Acknowledgement Sent by the DMA controller to acknowledge the end of each DMA burst or single transaction from the receive FIFO.

Synchronous To: pclk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High

dma_tx_req

 O

Transmit FIFO DMA Request - Asserted when the transmit FIFO requires service from the DMA Controller; that is, the transmit FIFO is at or below the watermark level.

0 - not requesting

1 - requesting

Software must set up the DMA controller with the number of words to be transferred when a request is made. When using the DW_ahb_dmac, this value is programmed in the SRC_MSIZE field of the CTLx register.

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

dma_rx_req

 O

Receive FIFO DMA Request - Asserted when the receive FIFO requires service from the DMA Controller; that is, the receive FIFO is at or above the watermark level.

0 - not requesting

1 - requesting

Software must set up the DMA controller with the number of words to be transferred when a request is made. When using the DW_ahb_dmac, this value is programmed in the DEST_MSIZE field of the CTLx register.

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

dma_tx_single

 O

DMA Transmit FIFO Single Signal This DMA status output informs the DMA Controller that there is at least one free entry in the transmit FIFO. This output does not request a DMA transfer.

0 - Transmit FIFO is full

1 - Transmit FIFO is not full

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

dma_rx_single

 O

DMA Receive FIFO Single Signal This DMA status output informs the DMA Controller that there is at least one valid data entry in the receive FIFO. This output does not request a DMA transfer.

0- Receive FIFO is empty

1- Receive FIFO is not empty

Synchronous To: pclk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High
