SDMA Interface Signals

Port Name I/O Description

dma_ack_tx

 I

DMA transmit acknowledgement sent by the DMA Controller to acknowledge the end of each DMA burst or single transaction to the transmit FIFO.

Synchronous To: dma_clk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High

dma_ack_rx

 I

DMA receive acknowledgement sent by the DMA controller to acknowledge the end of each DMA burst or single transaction from the receive FIFO. This should be asserted after the last data of the current transfer has been transferred to the destination indicating to the Target about the completion. It forms a handshake loop with dma_req_rx (de-asserts one apb clk cycle later).

Synchronous To: dma_clk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High

dma_req_tx

 O

Transmit FIFO DMA Request.

This signal is asserted when the transmit FIFO requires service from the DMA controller, that is, the transmit FIFO is at or below the threshold level.

Synchronous To: dma_clk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

dma_req_rx

 O

Receive FIFO DMA Request

Asserted when the receive FIFO requires service from the DMA Controller, that is, the receive FIFO is at or above the threshold level. An active level on dma_req_rx initiates a transaction request. The type of transaction, single or burst, is qualified by dma_single_rx. Once dma_req_rx is asserted, it must remain asserted until dma_ack_rx is asserted. When the Target determines that dma_ack_rx is asserted, it de-asserts dma_req_rx.

Synchronous To: dma_clk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

dma_single_tx

 O

DMA Transmit FIFO Single Signal

This DMA status output informs the DMA Controller that there is at least one free entry in the transmit FIFO.

Synchronous To: dma_clk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

dma_single_rx

 O

DMA Receive FIFO Single Signal

This DMA status output informs the DMA Controller that there is at least one valid data entry in the receive FIFO. If dma_single_rx is de-asserted in the same clock cycle as a rising edge on dma_req_rx, a burst transaction is requested by the peripheral. If asserted, the peripheral requests a single transaction. The dma_single_rx signal need only to be valid on the same clock cycle that dma_req_rx is generated. If this signal interfaces with AXI DMAC, extend it until dma_ack_rx is received. The controller also supports this extended dma_single_rx until dma_ack_rx is received where the parameter SDMA_RX_LAST_STROBE should be set. Thus, if the controller has be interfaced with AXI DMAC and the behavior of the dma_single_rx is required to be a pulse extended until dma_ack_rx is received, set the configuration parameter SDMA_RX_LAST_STROBE using set configuration parameter SDMA_RX_LAST_STROBE.

Synchronous To: dma_clk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

dma_last_rx

 O

Indicates last data to the external dma for rx channel. Applicable in only Slave mode of operation. The Target asserts dma_last_rx on the same cycle as dma_req_rx is asserted to signal that this transaction request is the last in the block. The block transfer is complete after this transaction is complete. If dma_single_rx is high in the same cycle, then the last transaction is a single transaction. If dma_single_rx is low in the same cycle, then the last transaction is a burst transaction. The dma_last_rx signal need only to be valid on the same clock cycle that dma_req_rx is generated. If this signal interfaces with AXI DMAC, extend it until dma_ack_rx is received. The controller also supports this extended dma_single_rx until dma_ack_rx is received where the parameter SDMA_RX_LAST_STROBE should be set. Thus, if the controller has to be interfaced with AXI DMAC and the behavior of the dma_single_rx is required to be a pulse extended until dma_ack_rx is received, set the configuration parameter SDMA_RX_LAST_STROBE using set configuration parameter SDMA_RX_LAST_STROBE.

Synchronous To: dma_clk

Registered: Yes

Power Domain: SINGLE_DOMAIN

Active State: High

Debug Interface Signals

Port Name I/O Description

debug_port[54:0]

 O

Debug port

On chip debug signal as follows:

  • Master Mode Debug Signals

debug[0] = Indicates if Master is the current master or not.

debug[1] = Indicates if SCL (scl_oe) is in Push-pull or Open-Drain Mode.

debug[2] = Indicates if SDA (sda_oe) is in Push-Pull or Open-Drain Mode.

debug[3] = Indicates SCL (scl_out) signal level.

debug[4] = Indicates SDA (sda_out) signal level.

debug[5] = Indicates if Command Queue is full.

debug[6] = Indicates if Command Queue is empty.

debug[7] = Indicates if Response Queue is full.

debug[8] = Indicates if Response Queue is empty.

debug[9] = Indicates if IBI Queue is full.

debug[10] = Indicates if IBI Queue is empty.

debug[11] = Indicates if Transmit Buffer is full.

debug[12] = Indicates if Transmit Buffer is empty.

debug[13] = Indicates Receive Buffer full.

debug[14] = Indicates Receive Buffer empty.

debug[20:15] = Indicates Current Master Transfer command.

debug[26:21] = Indicates Current Master Transfer command state.

debug[30:27] = Indicates Current Master Transfer command TID.

debug[31] = Indicates if the Master controller is in Idle state.

debug[32] = Indicates the Enable status of the controller.

debug[52:33] = Reserved

  • Slave Mode Debug Signals

debug[4:0] = Reserved

debug[5] = Indicates if Command Queue is full.

debug[6] = Indicates if Command Queue is empty.

debug[7] = Indicates if Response Queue is full.

debug[8] = Indicates if Response Queue is empty.

debug[10:9] = Reserved

debug[11] = Indicates if Transmit Buffer is full.

debug[12] = Indicates if Transmit Buffer is empty.

debug[13] = Indicates if Receive Buffer is full.

debug[14] = Indicates if Receive Buffer is empty.

debug[31:15] = Reserved

debug[32] = Indicates the Enable status of the controller.

debug[34:33] = Reserved

debug[39:35] = Indicates Slave Transfer State.

debug[43:40] = Indicates Slave Command TID.

debug[44] = Indicates slave reset complete

debug[51:45] = Indicates slave dynamic address

debug[52] = Indicates slave dynamic address valid

debug[53] = Indicates slave invalid transfer

Synchronous To: core_clk

Registered: No

Power Domain: SINGLE_DOMAIN

Active State: High
